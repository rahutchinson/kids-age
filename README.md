# Kids' Ages — TRMNL Plugin

Displays your children's ages on a TRMNL e-ink display. Shows age in years/months/days with an adaptive layout for 1–6 children.

---

## How it works

This is a **TRMNL third-party (shop) plugin**. Instead of TRMNL polling a JSON endpoint and rendering a Liquid template (the "private plugin" approach), a shop plugin works differently:

- **You own the server.** Your Node.js server handles OAuth, stores per-user settings, and renders the final HTML.
- **TRMNL calls your endpoints.** TRMNL drives the install flow, fetches your rendered HTML on a schedule, and links users to your settings page.
- **Each user has isolated data.** Multiple people can install the plugin; each gets their own children config stored server-side.

---

## The install flow (step by step)

### 1. User clicks "Install" on TRMNL

TRMNL redirects the user's browser to your `/install` endpoint with two query params:

```
GET /install?token=TEMP_CODE&installation_callback_url=https://usetrmnl.com/...
```

### 2. Your server exchanges the token

`/install` POSTs to `https://usetrmnl.com/oauth/token` with the temp code plus your client ID/secret. TRMNL returns a permanent `access_token` for this user. Your server saves the token to `data.json` and redirects the user back to TRMNL via `installation_callback_url`.

### 3. TRMNL sends a webhook

After the redirect, TRMNL POSTs to your `/webhooks/installed` endpoint:

```json
POST /webhooks/installed
Authorization: Bearer <access_token>

{ "user_uuid": "abc123", "plugin_setting_id": "xyz789", "timezone": "America/New_York" }
```

Your server uses the `access_token` from the header to look up the pending install, then creates a full user record in `data.json` keyed by `user_uuid`.

### 4. User configures their children

TRMNL shows a "Settings" link that opens:

```
GET /settings?plugin_setting_id=xyz789
```

Your server looks up the user by `plugin_setting_id`, renders an HTML form with their current children (if any), and lets them enter names + dates of birth. Submitting the form POSTs to `/settings` and saves the children array to `data.json`.

### 5. TRMNL fetches the display HTML

On a regular schedule (typically every 15–60 minutes), TRMNL POSTs to your `/markup` endpoint:

```
POST /markup
Authorization: Bearer <access_token>
Content-Type: application/x-www-form-urlencoded

user_uuid=abc123
```

Your server looks up the user's children, calculates each child's current age, renders the full HTML, and responds:

```json
{ "markup": "<style>...</style><div class=\"view view--full\">...</div>..." }
```

TRMNL puts that HTML directly on the e-ink display.

## Data storage (`data.json`)

All per-user state lives in a single JSON file on the server. It's git-ignored and auto-created on first use.

```json
{
  "users": {
    "abc123": {
      "access_token": "tok_...",
      "plugin_setting_id": "xyz789",
      "timezone": "America/New_York",
      "children": [
        { "name": "Emma", "dob": "2020-03-15" },
        { "name": "Liam", "dob": "2022-07-04" }
      ]
    }
  },
  "tokenIndex": {
    "tok_...": "abc123"
  },
  "settingIndex": {
    "xyz789": "abc123"
  }
}
```

The two index objects (`tokenIndex`, `settingIndex`) are reverse lookups so the server can find a user by token or by `plugin_setting_id` without scanning every record.

---

## Age calculation

The server calculates each child's exact age from their date of birth to today:

- **≥ 1 year old** → primary display is years (e.g. "5 years old"), secondary is "11 mo · 16 d"
- **≥ 1 month old** → primary display is months (e.g. "8 months old"), secondary is "16 days"
- **< 1 month old** → primary display is total days alive (e.g. "23 days old")

The `total_days` counter always shows the total days since birth (e.g. "2,180 days since birth"), regardless of which unit is primary.

---

## Display layouts

The rendered HTML uses TRMNL's CSS framework classes and adapts based on how many children there are:

| Children | Layout |
|----------|--------|
| 1 | Full screen, very large number centered |
| 2 | Side-by-side columns with vertical divider |
| 3 | Three equal columns with dividers |
| 4–6 | 2-column wrapping grid with border lines |

The HTML is generated by `templates/markup.js` → `renderMarkup(children)`. It's a direct port of the original Liquid template.

## Endpoints reference

| Method | Path | Called by | What it does |
|--------|------|-----------|--------------|
| GET | `/install` | TRMNL (redirect) | Exchanges OAuth token, redirects back |
| POST | `/webhooks/installed` | TRMNL | Creates user record after install |
| POST | `/webhooks/uninstalled` | TRMNL | Deletes user data on removal |
| GET | `/settings` | User's browser | Renders children config form |
| POST | `/settings` | User's browser | Saves children config |
| POST | `/markup` | TRMNL (periodic) | Returns rendered HTML |
| GET | `/health` | Anyone | Returns `{ status: "ok" }` |

## Setup

### 1. Deploy the server

You need a public HTTPS URL. Any Node.js host works:

- **Fly.io** — `fly launch` (Dockerfile is included)
- **Railway** / **Render** — connect the GitHub repo

### 2. Set environment variables

```env
PORT=3000
TRMNL_CLIENT_ID=your_plugin_client_id
TRMNL_CLIENT_SECRET=your_plugin_client_secret
BASE_URL=https://your-deployed-server.com
```

### 3. Register the plugin on TRMNL

Go to **usetrmnl.com/plugins/my/new**, set Strategy to **Third Party**, and fill in these five URLs:

| Field | Value |
|-------|-------|
| Install URL | `https://your-server.com/install` |
| Uninstall Webhook URL | `https://your-server.com/webhooks/uninstalled` |
| Installation Webhook URL | `https://your-server.com/webhooks/installed` |
| Settings URL | `https://your-server.com/settings` |
| Markup URL | `https://your-server.com/markup` |

Copy the generated **Client ID** and **Client Secret** into your `.env`. Enable **Debug Logs** while testing.

### 4. Install and configure

Install the plugin on your TRMNL account. Open the Settings link and enter your children's names and dates of birth (YYYY-MM-DD format). TRMNL will fetch the display on the next poll cycle.

## Testing locally

```bash
npm install
cp .env.example .env
# Fill in TRMNL_CLIENT_ID and TRMNL_CLIENT_SECRET
npm run dev
```

Expose your local server with [ngrok](https://ngrok.com/) or [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/), then register those URLs with TRMNL.

**Manually test `/markup`** after adding a user via settings:

```bash
# Find user_uuid and access_token in data.json, then:
curl -s -X POST http://localhost:3000/markup \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "user_uuid=<user_uuid>" | jq -r .markup
```
